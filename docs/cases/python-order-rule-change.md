# 案例 2：Python 订单折扣规则业务修改（初学者）

## 场景

现有规则：订单满 100 元打 9 折。  
新需求：
1. VIP 用户满 80 元打 85 折。
2. 普通用户保持满 100 元打 9 折。
3. 折扣后金额最低不低于 0 元。

## 第一步：新建任务

```bash
~/.codex/skills/codex-spec-zh/scripts/specflow.sh new "订单折扣规则调整"
```

默认会生成类似这样的目录：

```text
.codex/specflow/tasks/T20260327-001-订单折扣规则调整/
  meta.yaml
  任务说明.md
  代码库地图.md
  执行计划.md
  进度记录.md
  验证记录.md
  验收记录.md
```

如果你原来已有旧任务目录，例如 `T20260325-001-task/`，可以先执行：

```bash
~/.codex/skills/codex-spec-zh/scripts/specflow.sh localize T20260325-001
```

## 第二步：写 任务说明.md（示例）

1. 目标：支持 VIP 与普通用户两套门槛折扣。
2. 范围：`pricing/service.py` 与对应测试。
3. 非目标：不改数据库；不改下单流程接口。
4. 验收：
- VIP 订单 80 元以上按 85 折。
- 普通用户 100 元以上按 9 折。
- 所有订单最终金额 >= 0。

## 第三步：写 代码库地图.md（示例）

建议先找：
1. 折扣计算入口函数。
2. 订单金额和用户类型判定逻辑。
3. 对应单元测试文件和边界值测试位置。

建议记录：
1. 任务相关目录。
2. 关键入口文件。
3. 直接相关文件。
4. 潜在影响文件。
5. 搜索关键词（如 `discount`、`vip`、`price`、`order`）。

## 第四步：写 执行计划.md（示例 Step）

1. Step 1：新增或重构折扣计算函数。
2. Step 2：补齐单元测试（VIP/普通/边界值）。
3. Step 3：运行测试并修复失败。
4. Step 4：更新验收报告，并记录验证结果。

## 第五步：执行与验证

建议先在 `进度记录.md` 写清这一轮准备修改的函数、测试文件和不改边界，再执行命令。

如果验证命令较多，可以再单独写 `验证记录.md`，把输出、失败摘要和结论沉淀下来。

示例命令：

```bash
pytest -q
```

建议至少覆盖这些测试场景：
1. VIP，金额 79.99（不打折）。
2. VIP，金额 80.00（85 折）。
3. 普通用户，金额 99.99（不打折）。
4. 普通用户，金额 100.00（9 折）。
5. 异常输入（负值）时金额不低于 0。

## 第六步：归档

测试全部通过，并确认中文任务文档已经补齐后：

```bash
~/.codex/skills/codex-spec-zh/scripts/specflow.sh archive <TASK_ID> "折扣规则已上线"
```

## 你会学到什么

1. 如何把业务规则变更拆成可验证步骤。
2. 如何先定义验收再动代码。
3. 如何用最小改动实现业务演进，并把过程沉淀到中文任务文档里。
